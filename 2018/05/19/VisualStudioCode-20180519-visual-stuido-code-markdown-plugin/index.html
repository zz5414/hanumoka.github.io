마크다운 에디터용 비주얼스튜디오(VisualStudioCode) 플러그인(plugin) 설치

시작하기

나는 이 헥소블로그를 쓸때 비주얼스튜디오 코드를 사용한다.
이번에는 이 헥소블러그를 사용할때, 편할수도 있을 플러그인들을 정리해 보겠다.
플러그인에 대한 자세한 사용법은 정보페이지 링크를 참조하길 바란다.

언어와 상관없이 공통으로 쓸만한 플러그인

  • Rainbow Brackets
    코드작성시 중첩된 브라켓(브레이스)를 색으로 구별 해줘서 눈이 편하다.
  • vscode-icons
    프로젝트 폴더 내부의 파일과 폴더아이콘을 구별하기 쉬운 아이콘으로 보여준다.
  • Active File In StatusBar
    파일을 선택했을때 창의 하단에 해당 파일의 절대경로를 보여준다.
    Active File In StatusBar 정보 페이지
  • indent-rainbow
    탭이나 스페이스로 발생된 여백을 색으로 구별해준다.
  • Git History
    git history를 쉽게보여주고, 파일비교를 해준다.
    솔직히 이건 안써봐서 잘 모르겠지만 편해보여 설치했다.
    자세한 사용법은 아래를 참조하길 바란다.
    Git History 플러그인 정보페이지
  • Auto Close Tag
    HTML, XMl 등의 태그를 코딩할때 자동으로 닫아준다.
    Auto Close Tag 정보 페이지
  • Auto Rename Tag
    HTML, XML 태그를 수정할 경우, 자동으로 쌍으로 수정된다.
    Auto Rename Tag 정보 페이지

프론트엔드 개발시 쓸만한 플러그인

  • IntelliSense for CSS class names in HTML
    HTML코딩시 class에 css를 타이핑시 자동으로 찾아준다.
    IntelliSense for CSS class names in HTML 정보 페이지
  • Debugger for Chrome
    HTML페이지를 크롬으로 디버깅 해준다.
    Debugger for Chrome 정보 페이지
  • Beautify
    HTML, javascript, css, JSON 등등의 코드들을 예쁘게 보여준다.
    Beautify 정보페이지
  • ESLint
    javascript 문법체크, 코드스타일 체크등의 기능을 제공한다.
    ESLint 정보 페이지
  • jshint
    JSLint에서 파생되었으며 JSLint 보다 덜 업격한 javascript 정적 분석툴
    JSLint 정보 페이지
  • View In Browser
    HTML파일을 바로 브라우저로 볼수있게 해준다.
    View In Browser 정보 페이지
  • npm
    음 잘은 모르겠다. 나중에 node 개발할때 편해보여 일단 설치했다.
    npm 정보 페이지
  • npm Intellisense
    음 잘은 모르겠다. 나중에 node 개발할때 편해보여 일단 설치했다.
    npm Intellisense 정보 페이지

마크다운관련 플러그인

  • Markdown All in One
    다양한 단축키로 마크다운 작성시 편하다.
    Markdown All in One 정보 페이지
  • Markdown Shortcuts
    마크다운에서 테이블 만들때 편해보여 설치했다. 최초 테이블 작성시 편해보인다.
    Markdown Shortcuts 정보 페이지
  • Markdown TOC
    작성된 마크다운의 H 태그들을 자동으로 바로가기 목차로 만들어준다.
    Markdown TOC 정보 페이지
  • markdownlint
    마크다운을 lint 해준다.
    markdownlint 정보 페이지
  • Table Formatter
    마크다운에서 table 사용시 들쭉날쭉한 테이블을 예쁘게 정렬해준다. 이미 작성한 테이블을 수정할때 편해보인다.
    Table Formatter 정보 페이지
