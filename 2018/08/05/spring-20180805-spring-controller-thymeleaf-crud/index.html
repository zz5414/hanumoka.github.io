Spring controller와 thymeleaf뷰간 데이터 전달방식 정리(게시판 crud)

작성일 2018-08-05 Edited on 2020-04-16 In spring

실습환경

개발환경
windows10(64)
spring4
java1.8
tomcat9

들어가기

이 글은 spring 개인 프로젝트에 thymeleaf를 적용함에 있어서, spring controller와 thymeleaf간의 데이터 전달 방식을 정리하기 위한 글이다.
게시판 crud 예제를 통해서 설명할 것이다.

게시판 글 리스트 조회

1.controller

컨트롤러에서는 게시판글을 조회해서 글 vo를 List로 담아 model에 담는다.

2.list.html

위 html은 부트스트랩 테마때문에 좀 길긴 하다.
중요한 부분을 추리면 아래와 같다.

게시판 글 한개 조회화면

게시판 글 리스트에서 특정 글을 선택했을때 나오는 화면.

1.controller

특정 게시글 하나를 db에서 조회하여 model에 boardVO라는 이름으로 담아 넘긴다.

2.read.html

역시 부트스트랩 테마 때문에 좀 길다.
데이터가 맵핑되는 부분을 추려보겠다.
form 태그 내부를 보면 아래처럼 controller으로 부터 전달되는 데이터를 받는 부분이다.

게시글 수정 화면

1.controller

게시글 수정화면 (GET)이동 함수와, 실제 게시글 수정(POST)을 해주는 함수이다.

2.modify.html

게시글 수정화면은 read와 달리 모든 값을 다 서버로 전달해야 하므로 th:field로 모든 값을 처리한 것을 확인 할수 있다.

마무리 정리

일단 html 태그에 name 어트리뷰트가 불 필요 하다.
name 어트리뷰트를 사용할수 있지만, 햇갈리니 안쓰는게 좋아보인다.
서버에서 단순히 thymeleaf 뷰 html 파일에 값을 전달할때 th:value, th:field를 사용 할수 있다.
name 어트리뷰트가 없는경우 form 액션발생시 th:value는 컨트롤러에 값을 전달하지 않지만, th:field는 자동으로 값을 컨트롤러에 전달한다.
